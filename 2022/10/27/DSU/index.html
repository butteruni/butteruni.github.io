基础算法

并查集

引入

并查集，主要作于处理元素分组问题，一般操作为两个不同集合的合并或查找两元素是否在同一集合。如果觉得这种说法过于抽象，那不妨看看下面这题。

洛谷 P1551 >#### 题目背景 >若某个家族人员过于庞大，要判断两个是否是亲戚，确实还很不容易，现在给出某个亲戚关系图，求任意给出的两个人是否具有亲戚关系。 >#### 题目描述 >规定：\(x\) 和 \(y\) 是亲戚，\(y\) 和 \(z\) 是亲戚，那么 \(x\) 和 \(z\) 也是亲戚。如果 \(x\)，\(y\) 是亲戚，那么 \(x\) 的亲戚都是 \(y\) 的亲戚，\(y\) 的亲戚也都是 \(x\) 的亲戚。 >#### 输入格式 >第一行：三个整数n,m,p，（n<=5000,m<=5000,p<=5000），分别表示有n个人，m个亲戚关系，询问p对亲戚关系。 以下m行：每行两个数Mi，Mj，1<=Mi，Mj<=N，表示Mi和Mj具有亲戚关系。 接下来p行：每行两个数Pi，Pj，询问Pi和Pj是否具有亲戚关系。 >#### 输出格式 >P行，每行一个’Yes’或’No’。表示第i个询问的答案为“具有”或“不具有”亲戚关系。

如果互为亲戚的一组人看作同一集合中的元素，解决这个问题的实际操作也就只有两个：把互为亲戚的两人放入同一集合，查询两人是否在同一集合。于是想到使用并查集构造数据结构。

如何实现

在讨论具体代码之前，不妨回到问题本身，对于任何一个 “亲戚集合” 我们都可以找出其中 “最大" 的一个人，通过比较两个人所属 “亲戚集合” 的最大元素是否相同，就可以判断两个人是否属于同一 “亲戚集合”

而对于读入数据，我们可以先将每个人所属 “亲戚集合” 的 “最大元素” 定为自身，每次读到两个两个属于同一 “亲戚集合” 的两个人，则比较两人所属 “亲戚集合” 的 “最大元素” ，如果不同，就将其中一个人所属 “亲戚集合” 的 “最大元素” 指向另一 “亲戚集合” 的 “最大元素”

这样，就可以写出最基本的并查集代码。

初始化

int tree[1000]; 
void init(int n)
{
    tree[i] = i; //让每个人所属集合的最大元素为自身
}

查找最大元素

int find(int x)
{
    while(tree[x] != x)
    {
        x = tree[x]; //一直寻找，直到找到最大元素并返回
    }
    return x;
}

合并

void merge(int x,int y)
{
    int fx,fy;          //比较最大元素，不同则将其中一个最大元素和它所属的整个集合并入另一个集合
    fx = findx(x);
    fy = findx(y);
    if (fy != fx)
        relation[fy] = fx;        
}

写到这里，就完成了一个最简单的并查集代码。距离完成题目只剩下读入数据并比较。

简单的优化

路径压缩

对于最简单的并查集，数组 tree[i] 中存储的只是元素 i 指向的上一级元素，因此，随着合并次数的增加，寻找 “最大元素” 花费的时间也越来越多。我们希望每次访问数组都可以快速找到元素 i 所属集合的 “最大元素” ，而这也很好实现，只要在查询的过程中，把 沿途元素的上级元素设为 “最大元素” 即可。代码上使用递归实现

int find(int x)
{
    if(x == tree[x])
        return x;
    else
    {
        tree[x] = find(tree[x]);
        return tree[x];
    }
}

按秩合并/启发式合并

路径压缩被用于优化较深的并查集结构，同样，在合并两个并查集树时，我们希望高度低的树并入高度高的树，使生成的新树高度低些。

思路：对节点 i 有 rank[i] 表示以 i 为根节点的树的高度；

void init(int n)
{
    for(int i = 1; i <= n; ++i)
    {
        tree[i] = i;
        rank[i] = 1;
    }
}
int find(int x)
{
    while(tree[x] != x)
        x = tree[x];
    return x; 
}
